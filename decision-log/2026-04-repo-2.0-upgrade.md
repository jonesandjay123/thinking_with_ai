# 2026-04 Repo 2.0 Upgrade

## Decision

不另外建立新 repo，直接升級 `thinking_with_ai` 的資訊架構，讓它從「研究與專案堆疊」進化成更清楚的知識與產出中樞。

## Context

原本 `thinking_with_ai` 幾乎把大部分內容都放在 `projects/` 底下。這在內容少的時候很快，但隨著 repo 內的報告、方法論、專案、研究主題變多，開始出現一個問題：

- 什麼都叫 project
- 但其實內容性質並不相同
- 之後會越來越難判斷「新東西到底該放哪」

Jones 也明確意識到：

> 不想因為再開新 repo 而造成新的分散與困惑。

## Why not create a new repo?

因為新 repo 會帶來新的分類負擔：
- thinking_with_ai 放什麼？
- 新 repo 放什麼？
- 同一段對話衍生出的原則、研究、專案要拆去哪？

在目前這個階段，**分 repo 的成本大於收益**。

## New taxonomy

新增並明確化以下層次：

- `principles/`：長期有效的原則、方法論、思考框架
- `playbooks/`：可執行流程、操作指南
- `projects/`：具體主題研究、專案檔案、比較分析
- `decision-log/`：重要決策與方向轉折
- `ideas/`：尚未封裝的想法與 backlog
- `archive/`：已封存內容

## First concrete move

把 `thinking-building-compounding` 從 `projects/` 移到 `principles/`。

原因：
- 它不是單一專案
- 它更像長期方法論
- 它會成為未來很多內容的上位框架

## Expected benefit

這次升級的目標不是把 repo 弄得很複雜，而是讓未來每次新增內容時更容易判斷它的性質。

一句話總結：

> 不是多一個 repo，而是讓現有 repo 有更像樣的骨架。
