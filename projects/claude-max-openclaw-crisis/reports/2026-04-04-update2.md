## 📝 深度分析文章

### 1.「Claude Max Blocks OpenClaw. Now What?」（Ryan Eade / Focus Over Features）
**連結：** focusoverfeatures.substack.com

**核心觀點：**
- 這不是「封殺」，而是**計費解耦**：訂閱只涵蓋第一方產品
- Boris Cherny（Anthropic Claude Code 負責人）解釋：第一方工具針對 prompt cache 優化，OpenClaw 的 cache hit rate 低很多，每個 session 成本顯著更高
- Cherny 甚至向 OpenClaw 提交 PR 改善 cache hit rate — 這是善意的努力
- 但同時 Anthropic 剛推出 Channels、Dispatch、scheduled tasks — **與 OpenClaw 功能高度重疊**
- OpenClaw 創辦人剛被 OpenAI 挖走，時機「巧合」

**成本估算（轉 API 後）：**

| 使用等級 | 每日成本 | 每月成本 |
|---------|---------|---------|
| 輕量（個人助理，少量互動） | $5-15 | $150-450 |
| 中度（coding agent + heartbeat + email） | $20-50 | $600-1,500 |
| 重度（多 agent、24/7 自主運行） | $50-200+ | $1,500-6,000+ |

**建議策略：**
- **模型混用**：深度推理用 Opus，日常任務用 Gemini Flash，備援用其他 provider
- **Mega agent 已死**：拆分成小型專精 agent，各用最適合的模型
- **Mac Studio + 本地模型**混合架構：routine 任務跑本地，Opus 只做 coordination
- 精簡 system prompt、降低 heartbeat 頻率

### 2.「How to Survive OpenClaw on Codex」（telegrahi / Telegraphic）
**連結：** telegraphic.substack.com

**三週前就先行遷移到 Codex 的實戰經驗：**
- 前兩週是「噩夢」— 但多數問題出在 OpenClaw 本身（安全設定、sandbox、OOM）而非 Codex 模型
- 「你可能不會喜歡 claw 換了模型後的新人格，但與其抱怨，不如重新定義期望」
- 修復方法：讓 claw 讀 OpenClaw 原始碼來反向工程設定、定期監控 logs
- 推薦定時 log 檢查任務（每小時 journalctl 掃 error/warn）
- **關鍵心得：「如果你的 claw 只能跑在一個特定模型上，換了就崩潰，那它其實沒有真正 work。」**

## 💰 替代方案成本對比

### Anthropic API 定價（官方，2026-04-04）

| 模型 | Input (/MTok) | Output (/MTok) | Cache Read | Cache Write |
|------|--------------|----------------|------------|-------------|
| **Opus 4.6** | $5 | $25 | $0.50 | $6.25 |
| **Sonnet 4.6** | $3 | $15 | $0.30 | $3.75 |
| **Haiku 4.5** | $1 | $5 | $0.10 | $1.25 |

- Batch processing 享 50% 折扣

### Google Gemini / Vertex AI 定價

| 模型 | Input (/MTok) | Output (/MTok) | Cache Read |
|------|--------------|----------------|------------|
| **Gemini 3 Flash** | $0.50 | $3 | $0.05 |
| **Gemini 3.1 Flash-Lite** | $0.25 | $1.50 | $0.03 |
| **Gemini 3 Pro** | $2 | $12 | $0.20 |
| **Gemini 2.5 Flash** | $0.30 | $2.50 | $0.03 |
| **Gemini 2.5 Pro** | $1.25 | $10 | $0.13 |

- Google AI Studio 有免費額度（有 rate limit）
- Batch/Flex 模式再打 5 折

### OpenRouter 中轉

- 支援多家模型，OpenClaw 原生支援
- 加一層中間費用但提供統一帳單和 fallback 路由

### 成本對比情境：中度使用（~1M input + 200K output tokens/天）

| 方案 | 每日成本 | 每月成本 |
|------|---------|---------|
| Claude Opus 4.6 API | ~$10 | ~$300 |
| Claude Sonnet 4.6 API | ~$6 | ~$180 |
| Claude Haiku 4.5 API | ~$2 | ~$60 |
| Gemini 2.5 Flash | ~$0.80 | ~$24 |
| Gemini 3 Flash | ~$1.10 | ~$33 |
| Gemini 3.1 Flash-Lite | ~$0.55 | ~$17 |
| **混合策略**（Opus 推理 + Flash 日常） | ~$3-5 | ~$90-150 |

> **結論：混合策略每月 $90-150，接近原本 Max $100-200 的價位，但失去「吃到飽」的確定性。**

## 🛠️ 社群找到的應對方法

### ✅ 已驗證可行
1. **切換到 Anthropic API key** — 最簡單，改 config 即可，但按量付費
2. **切換到 Codex/OpenAI** — 可行但需重新調教人格（SOUL.md、AGENTS.md）
3. **混合模型策略** — 日常用 Gemini Flash，重要任務用 Opus API
4. **降低 heartbeat 頻率** — 從每 30 分鐘改為每天 2-4 次，大幅減少 token 消耗
5. **精簡 context window** — 砍 system prompt、減少每次 heartbeat 載入的上下文
6. **Extra Usage bundle** — 利用 Anthropic 提供的 30% 預購折扣
7. **一次性 credit** — 記得在 4/17 前兌換等值月費的 credit

### 🔬 進階方案
8. **Mac Studio + 本地模型** — routine 任務用本地 LLM，Opus 做 coordination
9. **OpenClaw log 監控** — 設定定時任務掃 gateway logs，提早發現問題
10. **ClawBuddy skill** — 社群開發的自我診斷技能（clawhub.ai/musketyr/clawbuddy-hatchling）

### ❌ 未發現的
- 沒有繞過新政策繼續用訂閱的方法
- 沒有官方遷移工具或腳本
- 沒有任何 provider 提供等值的「吃到飽」agent 方案
