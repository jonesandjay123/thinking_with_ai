# AI Opportunity Radar 🛰️

> 每天自動偵測世界的結構性低效率，產出可行的商業機會。
> 「找商機的機器本身就是商機。」

## 核心理念

不直接賭某個產品，而是建一台**持續產生商機線索的引擎**。

靈感來源：非洲可洗衛生棉 — 不是發明新科技，而是發現**結構型機會**：
1. 巨大但被忽略的需求
2. 本地已有資源
3. 只差一個設計 / 組織 / pipeline
4. 一旦建立就能 scale

## 降維打擊定義

- **不是**技術圈裡的 SaaS 互卷
- **而是**用 AI 加速學會任何領域的 know-how，去做傳統需要多年經驗的生意
- 賣給不懂 AI 的普通人
- 前台是人的溫度，後台是 AI 效率
- 滿足人性需求（吃/情感/社交/虛榮）— AI 長遠無法取代的領域

## 三層架構

### Layer 1: 訊號蒐集層

每日從多個來源抓取「世界的 bug」：

| 類別 | 來源 | 抓什麼 |
|------|------|--------|
| 抱怨 | Reddit, PTT, Amazon reviews | "why is there no…", "so annoying", "I wish…" |
| 趨勢 | TikTok, 小紅書, Product Hunt, Google Trends | 正在爆但還沒飽和的東西 |
| 結構 | NGO reports, World Bank, 城市論壇 | 被制度卡住的市場 |
| 價差 | 跨國電商比價, 二手平台 | 同商品在不同市場的價差 |
| 文化差 | 各語言社群 | 亞洲火了但西方還沒進來的東西 |

**MVP 先做 2 個來源：Reddit + 小紅書/PTT**

### Layer 2: AI 判讀層

每條訊號打分：

| 維度 | 說明 |
|------|------|
| 需求強度 | 這是不是人真的痛 (1-5) |
| 痛苦頻率 | 每天發生 vs 偶發 (1-5) |
| 付費意願 | 人會不會掏錢 (1-5) |
| 自動化潛力 | 能不能搭成流水線 (1-5) |
| 可行距離 | 和 Jones 的能力/資源距離 (1-5) |
| 競爭密度 | 紅海 vs 藍海 (1-5) |
| 跨國套利 | 能不能從一地搬到另一地 (1-5) |
| 實體化可能 | 能不能包成有形商品/服務 (1-5) |

**強制輸出三件事：**
1. 這個痛點的本質是什麼
2. 現成資源有哪些可以借力
3. 可以怎麼最小成本驗證

**額外偵測維度（GPT 建議）：**

| 維度 | 抓什麼 | 為什麼值錢 |
|------|--------|-----------|
| 🔧 Workaround Detected | 人用奇怪方式解決問題（膠帶貼 webcam → webcam cover） | 需求存在但還沒被產品化 |
| 😤 Paying But Angry | 人已經在付錢但很不爽（Ticketmaster 手續費、房仲佣金） | 市場已驗證，只是 solution 很爛 |
| 🎯 Jones Leverage | 這件事 Jones 是不是特別有優勢（AI/跨文化/tech/海外） | 避免「好機會但不是你的機會」 |

### Layer 3: 機會生成層

每日報告模板：

```markdown
# Daily Opportunity Brief — YYYY-MM-DD

## 🔴 今日最值得注意的訊號
一句話摘要 + 來源連結

## 🧠 背後的人性 / 結構問題
- 懶？羞恥？孤獨？時間不夠？
- 跨國資訊差？本地資源沒被用起來？

## 🔧 可行 Business Pipeline
用流程寫：
> 問題 → 解法 → 最小可行產品 → 規模化路徑

## ✅ 最小驗證方法
- 發 3 篇貼文測詢問量？
- 做假 landing page 測點擊？
- 直接聯絡 10 個潛在客戶？
- 用 AI 先手工跑 3 單？

## ⚙️ 自動化潛力
未來能不能變成：
- 內容流水線 / 代辦流水線 / 仲介流水線 / 電商流水線？

## 📊 評分
| 維度 | 分數 |
|------|------|
| 需求強度 | /5 |
| ... | ... |
| **總分** | /40 |
```

## 長期觀察：機會聚類

跑一段時間後，機會會開始**聚類**。例如很多都在「孤獨/情緒」或「跨國套利」或「local service inefficiency」。那時候就會浮現 **Jones 的主戰場** — 不是一開始就決定，而是看了一千個問題後，發現自己最有感的是哪一類。
