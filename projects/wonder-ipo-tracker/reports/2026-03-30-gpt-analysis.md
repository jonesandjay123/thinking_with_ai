## 二、關鍵術語白話解釋

| 術語 | 白話 |
|------|------|
| **FMV** | 公司請會計師估的「現在一股大概值多少」，不是市場價 |
| **ITM (In The Money)** | 你的券已經有價值（行使價 < FMV） |
| **NSO** | 員工選擇權的「普通版」，exercise 當下就要繳稅 |
| **ISO** | 員工選擇權的「優惠版」，稅比較友善（你的不是這個） |
| **AMT** | 替代最低稅負——ISO 才有這個問題，NSO 沒有 |
| **Long-term Capital Gain** | 持有超過一年再賣，稅率較低 |
| **Lock-up** | IPO 後內部人被禁止賣股票的期間（通常 180 天） |
| **Cashless Exercise** | 買股票的同時立刻賣掉，不用先掏現金（IPO 後才可能有） |
| **Right of First Refusal** | 你想賣股票時，公司有權優先用同樣價格買走 |

## 三、NSO 的稅務機制（最重要）

```
Exercise 那一刻：
  (FMV - $1.10) × 股數 = 被當成「普通收入」課稅
  → 還沒賣就要繳稅！

之後賣掉時：
  (賣價 - FMV) = Capital Gain
  → 持有 > 1年 = 長期稅率（低）
  → 持有 < 1年 = 短期稅率（高）
```

**⚠️ 最大風險**：如果 exercise 後公司股價崩到 $0，你已經繳的稅拿不回來。

## 四、180 天 Lock-up 機制

- **這是業界標準**，不是 Wonder 特有（Uber、Airbnb、Snowflake 全都有）
- **目的**：防止 IPO 當天 insider 全部倒貨導致股價崩盤
- **適用對象**：員工、創辦人、VC 投資人——全部一起鎖
- **但大股東有更多工具**：可能有提前釋放、secondary sale、對沖等
- **核心邏輯**：如果你在 IPO 前已經 exercise（已是股東），就會被鎖 180 天

**關鍵不確定性**：如果等 IPO 後才 exercise，是否仍被鎖？這取決於 IPO 實務安排，合約本身沒寫死。

## 五、Jones 的核心直覺與 GPT 的校正

### Jones 的判斷
- 這公司「行銷大於實質」，食物跟服務都很爛
- 感覺是衝 IPO 套現型，上市後可能被看破手腳
- 作為前員工，直覺上不信任這間公司的基本面

### GPT 的校正
- **部分認同**：消費者負面評價、燒錢擴張的 pattern 確實像「story > fundamentals」
- **但提醒不要過度確定**：Marc Lore（CEO）的紀錄是「很會設計 exit」（Diapers.com → Amazon、Jet.com → Walmart $33 億）
- **關鍵區分**：
  - 問題 A「這是不是好公司」→ 可能不是
  - 問題 B「這股票能不能讓你賺錢」→ 跟 A 沒有直接關係
- **Uber/Robinhood vs Wonder 的差異**：前者即使跌，品牌、市場、流動性都在，$1.10 成本怎樣都划算。Wonder 的風險是「市場可能根本不長期相信這個故事」

## 七、IPO + Lock-up 典型劇本（待深入研究）

| 階段 | 常見情境 |
|------|---------|
| IPO 前 | 公司拼命美化數字、講故事 |
| IPO 當天/首週 | 散戶 FOMO 推高股價 |
| 前 1-3 個月 | Hype 維持 |
| 180 天 Lock-up 到期 | Insider 大量拋售，供給暴增 |
| 到期後 1-3 個月 | 市場開始「看真相」，股價壓力最大 |

### 需要 Jarvis 後續研究的案例
- **翻車型**：WeWork（IPO 失敗）、Uber（lock-up 後跌）、Robinhood（散戶 hype 反噬）
- **中性型**：Airbnb（有基本面撐住）
- **成功型**：Snowflake（真正強的公司不怕 lock-up）
- **重點**：區分「跌了但仍是好資產」vs「market 開始看穿故事」
